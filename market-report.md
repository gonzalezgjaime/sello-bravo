## Data provenance & caveats
- **REAL** = live Mercado Libre (MX / site MLM) search data. **EST** = estimated (no real measured data for that niche).
- Mercado Libre's public API now requires an OAuth app token and typically omits `sold_quantity`; when demand is unmeasured it is treated as neutral (not zero, not max), so a blank demand does not inflate or sink a niche.
- Mixed granularity: **competition** is the whole-marketplace listing count (`paging.total`), while **demand, price, and top-seller share** come only from the sampled top `limit` results. Top-seller share is sampled concentration, not true market share.
- Amazon MX is EST-only until an SP-API seller account is connected; it does not yet contribute to scores.
- Missing or uninformative (all-equal) signals normalize to a neutral 0.5; only genuinely-varying measured values span the full range. Scores are relative to the niches in this run; adding/removing niches re-scales them.
